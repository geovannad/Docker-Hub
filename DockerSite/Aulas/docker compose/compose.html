Conceito de Docker Compose

Conceito e definição

Docker Compose é uma ferramenta que facilita a definição e a execução de aplicações multi-container no Docker. Com Docker Compose, você pode usar um arquivo YAML para configurar os serviços da sua aplicação. Esse arquivo, chamado docker-compose.yml, permite definir os contêineres, redes, volumes e outras configurações necessárias para sua aplicação funcionar.

Conceitos Principais do Docker Compose

Serviços (Services):

Um serviço em Docker Compose é uma definição de um contêiner Docker. Cada serviço representa um contêiner que será gerenciado pelo Docker Compose. Por exemplo, você pode ter serviços separados para um banco de dados, um servidor web, um serviço de cache, etc.

Volumes:

Volumes são usados para persistir dados entre reinicializações dos contêineres. Com volumes, os dados gerados e usados pelos contêineres podem ser armazenados de maneira persistente fora do ciclo de vida dos próprios contêineres.

Redes:

Docker Compose permite definir redes personalizadas para permitir que os serviços se comuniquem entre si de forma isolada e segura. Por padrão, Docker Compose cria uma rede isolada para cada projeto, mas você pode configurar isso conforme suas necessidades.

Builds:

Você pode especificar como os contêineres devem ser construídos diretamente no docker-compose.yml. Isso pode incluir a especificação de um Dockerfile, contexto de build e outras opções de construção.

Exemplo de construção

version: '3'

services:

web:

image: nginx:latest

ports:

- "80:80"

database:

image: postgres:latest

environment:

POSTGRES_PASSWORD: example


Nesse exemplo, definimos dois serviços: web e database. O serviço web usa a imagem nginx:latest e mapeia a porta 80 do contêiner para a porta 80 do host. O serviço database usa a imagem postgres:latest e define uma variável de ambiente POSTGRES_PASSWORD.

Comandos Básicos do Docker Compose

docker-compose up:

Inicializa e executa todos os serviços definidos no arquivo docker-compose.yml. Se os contêineres já existirem, eles serão reiniciados, caso contrário, serão criados.

docker-compose down:

Para e remove todos os contêineres, redes e volumes definidos no arquivo docker-compose.yml.

docker-compose build:

Constrói ou reconstrói as imagens dos serviços definidos.

docker-compose ps:

Lista os contêineres em execução gerenciados pelo Docker Compose.

docker-compose logs:

Exibe os logs de saída dos serviços em execução.

Exemplo de uso

O Docker Compose é como um conjunto de plantas mobiliadas para construir um bairro inteiro. Ele não define apenas uma única casa (um único serviço), mas várias casas (múltiplos serviços) e como elas se relacionam entre si. Você pode pensar no docker-compose.yml como o projeto do bairro, especificando cada casa, suas conexões (como estradas e redes) e onde ficam os serviços compartilhados (como parques ou subestações de energia).
